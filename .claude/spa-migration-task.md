Write `web/loader.js` as a vanilla JavaScript file (no bundler, no ES module syntax that requires a bundler — use standard script-tag-compatible JS). This file runs as a plain `<script>` before any JSX is loaded. It must populate all `window.BD_*` globals and then boot React.

```js

  name:    name,
  hook:    /* first sentence of description: split on '. ', take [0] + '.' */,
  tags:    tags,
  drive:   drive,
  price:   /* tier: 0–15 → "$", 16–30 → "$$", 31+ → "$$$", null/missing → "$" */,
  rating:  rating,

}
```

4. Assign result array to `window.BD_ACTIVITIES`.

### B. Schedule → `window.BD_SCHEDULE`

1. `fetch('schedule.json')`
2. Group events by date. Build an array of day objects:

```js
{

  day:    "Monday",
```

3. Each event object:

```js

  type:       event.type || "open",
  sub:        null,
  activityId: event.catalogRef || null
}
```

4. Assign to `window.BD_SCHEDULE`.

### C. People → `window.BD_PEOPLE`

### D. Supabase hydration

1. Read `web/js/picks.js` to extract `SUPABASE_URL` and `SUPABASE_ANON_KEY`. Embed inline in `loader.js`.
2. Load Supabase JS v2 CDN client (added in Phase 6 index.html).

5. Store client as `window.BD_SUPABASE`.

## PHASE 4 — SUPABASE WRITE-BACK (modify `web/Shell.jsx`)

Wire `toggleWish` and `toggleCommit` in the appReducer to persist to Supabase after the optimistic in-memory update:

**toggleWish:**
```js
const supaUserId = userId.charAt(0).toUpperCase() + userId.slice(1);
if (adding) {
  window.BD_SUPABASE.from('picks').upsert({ user_id: supaUserId, slug: activityId, state: 'wishlist' });
} else {
  window.BD_SUPABASE.from('picks').delete().match({ user_id: supaUserId, slug: activityId });
}
```

**toggleCommit:** same pattern, `state: 'committing'`.

Do not await on the UI path. Fire and forget; the optimistic update handles immediate UI response.
