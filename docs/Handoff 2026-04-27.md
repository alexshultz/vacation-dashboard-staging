## Architecture Decision Pending -- Schedule Editor

Alex wants a schedule editor page where he can assign startTime and travelMinutes to events and see the changes live for all family members.

**Decided architecture:** Option A -- GitHub API writes
- Standalone `editor.html` (admin-only, not linked from family nav)
- Loads schedule.json from the repo
- Alex edits startTime (time picker), travelMinutes (number field), priority per event
- "Save" button commits changes to schedule.json via GitHub API using a personal access token stored in localStorage
- GitHub Pages auto-rebuilds (~60 seconds) and everyone sees the update
- No server, no Supabase required for this feature

**NOT decided yet:**
- startTime as time picker vs free text (Alex was asked, session ended before answering)
- Whether to pre-populate travelMinutes from the attractions catalog
- Exact UI layout of the editor

**Note saved to memory:** Admin screen should include a control to adjust INITIAL_VISIBLE dynamically.
